## Codebase Patterns
- Chrome extension project using Vite + React + TypeScript
- State management via React Context (src/store/index.tsx)
- No test framework configured; quality checks = `npx tsc --noEmit`
- date-fns used for date formatting with locale support (zhCN, enUS, ja)
- `initReactI18next` plugin auto-binds i18n to React — no I18nextProvider wrapper needed when using default instance
- `noUnusedLocals` and `noUnusedParameters` are enabled in tsconfig — be careful not to import unused things
- Background service worker (src/background/index.ts) cannot import React — use `i18next` directly with inline locale imports, NOT `../i18n` (which pulls in react-i18next)
- Non-React utility files (date.ts, report.ts) can import `i18n` from `../i18n` since they run in the React context
- Template i18n uses `{{t:key.path}}` syntax to distinguish from data placeholders `{{key}}` in weekly report template
- `getDateFnsLocale()` exported from date.ts — reuse in report.ts for locale-aware date formatting
- `PRIORITY_CONFIG.label` is an i18n key string (e.g. 'task.priorityHigh'), not a translated string — components must call `t()` on it
