## Codebase Patterns
- Vite `base: './'` is required for Chrome extensions so asset paths are relative (not absolute `/`)
- `public/` directory contents are copied to `dist/` root by Vite, so Chrome extension paths should reference them without `public/` prefix (e.g., `icons/icon-128.png` not `public/icons/icon-128.png`)
- Manifest.json is not auto-copied by Vite; use a `writeBundle` plugin hook to copy it
- Background service worker entry is configured as a separate Rollup input with custom `entryFileNames` to output as `background.js` at dist root
- `npm run build` runs `tsc && vite build` — typecheck first, then build
- All chrome.storage.local access should go through `src/utils/storage.ts` — never call chrome.storage directly from UI components
- Shared Framer Motion animation variants live in `src/components/ui/animations.ts` — use named states (initial/animate/exit) with the `variants` prop
- `cardIn` variant includes both entrance (fade up) and exit (slide left) animations — use it for list items that need both
- Use `staggerContainer` + `staggerItem` variants for page load animations; don't use manual delay values
- Form components in modals should use `fadeInUp` (spring entrance from bottom) rather than `cardIn`
- Tag selection logic should use `src/hooks/useTagSelection.ts` hook; tag selection UI should use `src/components/ui/TagSelector.tsx`
- Design theme colors are defined as CSS custom properties in `src/styles/globals.css` under `:root` — Tailwind config references these via `var(--color-*)`, SVG and raw CSS should use the same variables
