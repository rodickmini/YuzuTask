# PRD: 电子宠物互动增强

## 背景

当前 PetMascot 组件仅有"换一只"按钮切换宠物，缺乏与用户的深度互动。希望通过鼠标交互和任务奖励机制，让宠物成为用户的"工作搭子"，提升产品趣味性和用户粘性。

## 现状

- 4 个 Lottie 动画宠物（3 猫 1 狗），仅支持随机切换
- 无点击/悬浮交互反馈
- 无奖励/积分系统
- 任务完成和番茄钟与宠物完全独立

---

## 方案一：鼠标悬浮 & 点击互动（优先级：P1）

### 悬浮交互
- 鼠标悬浮宠物区域时，气泡文字切换为互动台词（如"摸摸我~"、"嘿嘿~"）
- 宠物轻微放大 + 微弹动画（Framer Motion `whileHover`）
- 悬浮时宠物周围出现小星星/爱心粒子（可选，用 CSS 或 canvas）

### 点击交互
- 点击宠物本体 → 随机触发一种反应：
  - 开心跳跃动画
  - 爱心粒子飘出
  - 随机台词气泡（"喵~"、"汪汪！"、"再摸就生气了哦"等）
- 冷却机制：连续快速点击时台词变化，避免重复

### 技术要点
- 纯前端，无需状态持久化
- 利用现有 Framer Motion + Lottie 能力
- 气泡台词用数组随机抽取

---

## 方案二：猫粮/狗粮奖励系统（优先级：P1）

### 获取粮食
| 行为 | 奖励 |
|------|------|
| 完成一个任务 | +1 粮食 |
| 完成一个番茄钟 | +2 粮食 |
| 每日首次打开 | +3 粮食（签到） |

### 消耗粮食
- 点击"投喂"按钮 → 消耗 1 粮食
- 触发进食动画 + 宠物开心反应
- 气泡文字变化："好吃！"、"再来一碗！"

### 饱腹度 & 心情
- 饱腹度 0-100，投喂 +20，每小时自然下降 5
- 饱腹度影响气泡文字：
  - >80：开心台词
  - 30-80：普通台词
  - <30：饿了台词（"肚子咕咕叫..."）
- 心情值 = 饱腹度 + 互动加分（每次点击/悬浮 +1，每日上限 10）

### 持久化
- 粮食数量、饱腹度、心情值存 localStorage
- 记录最后更新时间，用于计算离线期间饱腹度衰减

### UI 设计
- 宠物区域下方新增：粮食数量显示 + "投喂"按钮
- 粮食用小图标表示（🐟 猫粮 / 🦴 狗粮，根据当前宠物类型）
- 投喂时有飘字动画 "+1 饱腹度"

---

## 方案三：进阶玩法（优先级：P2，远期）

- 宠物等级系统：投喂和互动积累经验值，宠物升级
- 不同宠物偏好不同食物（猫咪爱吃鱼，狗狗爱吃骨头）
- 心情系统联动番茄钟：心情高时显示鼓励文字，心情低时提醒休息
- 宠物换装：用粮食兑换装饰品（帽子、围巾等）
- 成就系统：连续投喂 7 天、累计完成 50 个任务等

---

## 涉及文件

- `src/components/ui/PetMascot.tsx` — 主要修改
- `src/store/index.tsx` — 新增粮食/心情相关 state 和 action
- `src/hooks/usePomodoro.ts` — 番茄钟完成时触发粮食奖励
- `src/components/TaskList.tsx` — 任务完成时触发粮食奖励
- `src/utils/petStorage.ts` — 新建，宠物状态持久化工具

## 实施建议

1. 先做方案一（鼠标互动），快速上线，提升即时的趣味感
2. 再做方案二（奖励系统），需要改 store 和多个组件，工作量较大
3. 方案三根据用户反馈决定是否推进
